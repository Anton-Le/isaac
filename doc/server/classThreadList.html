Detailed Description

template<typename T>
class ThreadList< T >

This is a special list, where one (!) thread can push back while one (!) can pop_front with only one mutex needed and (hopefully) not too much busy waiting However there are some rules:

  • As said: Only one thread can only pop_front and remove
  • As said: Only one thread can use push_back
  • Only the thread, which uses pop_front, may also iterate over the list
  • Iterating should only use getFront and ->next
